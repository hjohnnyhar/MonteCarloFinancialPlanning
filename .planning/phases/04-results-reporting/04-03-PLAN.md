# Plan 04-03: PDF report generation with @react-pdf/renderer

**Phase:** 04
**Sequence:** 03 of 03
**Requirements:** RSLT-04
**Depends on:** 04-02

## Goal

Add a download button to the simulation page that generates a client-side PDF report containing the headline score, goal breakdown, recommendations, plan summary, and year-by-year projection table.

## Context

Per D-13, the PDF is generated client-side using `@react-pdf/renderer` (no server-side puppeteer). Per D-14, the PDF contains: headline score + tier label, goal-by-goal breakdown (probability + funding gap), all 3 recommendations with amounts, plan summary (age, income, savings rate, total assets), and year-by-year projection table. The simulation page (updated in 04-02) already has all this data available in `SimulationResults` and `FinancialPlan` state. `formatCurrency` and `formatGoalType` are now in `src/lib/formatters.ts`. The PDF component uses `@react-pdf/renderer`'s `Document`, `Page`, `View`, `Text`, `StyleSheet` — this is NOT regular React DOM, so Tailwind classes cannot be used. Styles must use react-pdf's `StyleSheet.create()` with CSS-in-JS objects.

## Tasks

### Task 1: Install @react-pdf/renderer and create PDF document component
- [ ] Run `npm install @react-pdf/renderer` to add the dependency
- [ ] Create `src/components/pdf/FinancialPlanReport.tsx` — a react-pdf `<Document>` component that receives `{ plan: FinancialPlan, results: SimulationResults }` as props
- [ ] Page 1: Title "Financial Plan Report" + generation date. Headline section: large probability percentage, tier label and color (green/amber/red mapped to hex), explanation sentence. Plan summary section: current age, annual income, savings rate as %, total investable assets, risk tolerance level
- [ ] Page 1 continued or Page 2: Goal Breakdown section — table with columns: Goal Type, Target Amount, Probability, Funding Gap. One row per goal. Color-code probability text (green >= 80%, amber >= 60%, red < 60%)
- [ ] Next section: Recommendations — numbered list of all 3 recommendations with summary text, current vs suggested values
- [ ] Final section: Year-by-Year Projection table — columns: Year, Age, Portfolio Value, Savings, Withdrawals, Milestone. Highlight milestone rows. This may span multiple pages; react-pdf handles page breaks automatically via `wrap` prop
- [ ] Use `StyleSheet.create()` for all styles. Font: Helvetica (built into react-pdf, no font registration needed). Colors: use hex equivalents of the app's color tokens (green-600: #16a34a, amber-600: #d97706, red-600: #dc2626, gray-900: #111827, gray-500: #6b7280, blue-600: #2563eb). Page margins: 40pt. Section spacing: 20pt
- [ ] Create a currency formatter that works without Intl (react-pdf runs in a worker context where Intl may not be available) — simple `$X,XXX` formatting function

### Task 2: Add download button to simulation page
- [ ] Create `src/components/simulation/DownloadPdfButton.tsx` — receives `{ plan: FinancialPlan, results: SimulationResults }` props. Uses `@react-pdf/renderer`'s `pdf()` function to generate a blob on click, then triggers a browser download with filename `financial-plan-report.pdf`. Button shows "Download PDF Report" with a loading state ("Generating...") while the PDF is being created. Style as a secondary button: `rounded-md border border-gray-300 bg-white px-4 py-2.5 text-sm font-semibold text-gray-700 hover:bg-gray-50`
- [ ] Import `DownloadPdfButton` in `page.tsx` and render it at the top of the results section, right-aligned in the header area next to the page title. Only visible when `results` and `plan` are both loaded
- [ ] Handle the dynamic import pattern: `@react-pdf/renderer` is client-only and large. Use Next.js `dynamic(() => import(...), { ssr: false })` or lazy import inside the click handler to avoid SSR issues. The `pdf()` function approach (imperative blob generation) is preferred over `<PDFDownloadLink>` for better loading UX control
- [ ] Test error handling: if PDF generation fails, show an inline error message below the button ("Failed to generate PDF. Please try again.") instead of crashing
