# Plan 04-01: Extend simulation engine with recommendations and yearly projection

**Phase:** 04
**Sequence:** 01 of 03
**Requirements:** RSLT-01, RSLT-02, RSLT-03
**Depends on:** none

## Goal

Extend the simulation engine to compute rule-based recommendations with specific dollar amounts and extract a year-by-year median projection path, then update types and the API route to return this data.

## Context

The simulation engine (`src/lib/simulation.ts`) currently runs 10,000 Monte Carlo iterations and returns `SimulationResults` with `overallProbability`, `goalResults[]`, and `assumptions`. Phase 4 needs three additions: (1) a `recommendations[]` array with computed dollar amounts for 4 levers (per D-05, D-06, D-07, D-08), (2) a `yearlyProjection[]` with median-path data per year (per D-09, D-10, D-12), and (3) score tier metadata for the headline (per D-03, D-04). The API route (`/api/simulate`) returns `SimulationResults` directly, so extending the type extends the API response automatically (per D-08).

## Tasks

### Task 1: Extend types with Recommendation and YearlySnapshot
- [ ] Add `Recommendation` interface to `src/lib/types.ts`: `{ lever: 'savings_increase' | 'retirement_delay' | 'spending_reduction' | 'goal_reduction'; summary: string; currentValue: number; suggestedValue: number; projectedScore: number; }`
- [ ] Add `YearlySnapshot` interface to `src/lib/types.ts`: `{ year: number; age: number; portfolioValue: number; annualSavings: number; annualWithdrawal: number; goalMilestone: string | null; }`
- [ ] Add `ScoreTier` type to `src/lib/types.ts`: `{ label: 'Strong plan' | 'On track' | 'At risk'; color: 'green' | 'amber' | 'red'; }`
- [ ] Extend `SimulationResults` interface with three new optional fields: `recommendations?: Recommendation[]`, `yearlyProjection?: YearlySnapshot[]`, `scoreTier?: ScoreTier` — optional so existing tests do not break

### Task 2: Build recommendations engine and median-path extraction in simulation.ts
- [ ] Add `computeScoreTier(probability: number): ScoreTier` — thresholds: >=0.8 green/"Strong plan", >=0.6 amber/"On track", <0.6 red/"At risk" (per D-04)
- [ ] Add `computeRecommendations(plan, results, overrides?): Recommendation[]` function that evaluates 4 levers (per D-06): (1) savings rate increase — re-run simulation with incremented savings rate to find what reaches 90%, (2) retirement age delay — only if plan has retirement goal, try +1/+2/+3 years, (3) monthly spending reduction — reduce discretionary spending, (4) goal target reduction — reduce the worst-funded goal's target. Each recommendation includes the specific dollar/year amount and the projected new score. Return top 3 sorted by impact (per D-07). Use a lightweight re-simulation approach: run 2,000 iterations per lever (not full 10,000) for performance
- [ ] Add `extractMedianPath(plan, overrides?): YearlySnapshot[]` function that runs a single median-extraction pass: run 1,000 iterations, collect portfolio value at each year, take the median across iterations for each year. Include goal milestone markers at years when goals are due (per D-10)
- [ ] Update `runSimulation()` to call `computeScoreTier()`, `computeRecommendations()`, and `extractMedianPath()` after the main MC loop, adding results to the returned `SimulationResults` object
- [ ] Ensure total simulation time stays under 5 seconds (the main loop is 10k iterations; recommendations add ~8k iterations across 4 levers at 2k each; median path adds 1k — total ~19k which should complete in <3s based on current performance)

### Task 3: Add unit tests for recommendations and median path
- [ ] Create `src/lib/__tests__/recommendations.test.ts` with tests: (1) `computeScoreTier` returns correct tier for each threshold boundary, (2) `computeRecommendations` returns exactly 3 recommendations sorted by impact, (3) retirement delay recommendation is absent when plan has no retirement goal, (4) savings increase recommendation shows a higher savings rate value than current
- [ ] Create `src/lib/__tests__/yearly-projection.test.ts` with tests: (1) `extractMedianPath` returns one snapshot per year for the plan horizon, (2) each snapshot has all required fields (year, age, portfolioValue, annualSavings, annualWithdrawal, goalMilestone), (3) goal milestone markers appear at the correct year
- [ ] All existing simulation tests still pass
