Per D-10: When on `/interview`, Sidebar renders wizard step list instead of normal nav links. Per D-11: Step labels remain clickable after completion for edit navigation (INT-07). Per D-05: Free back navigation.

**Why wizardStore instead of React context:** The Sidebar is rendered by AppShell (parent layout), while the InterviewPage is a child route. A React context provider in the interview page would NOT be visible to the Sidebar because the Sidebar is rendered outside the page's component tree. A lightweight external pub/sub store with `useSyncExternalStore` solves this cleanly — both Sidebar and InterviewPage can read/write the same state regardless of component tree position.

1. **Create `src/lib/wizardStore.ts`** — a lightweight pub/sub store for wizard state shared between Sidebar and InterviewPage:
   ```typescript
   type Listener = () => void;
   let _stepIndex = 0;
   let _completedSteps: number[] = [];
   const _listeners = new Set<Listener>();

   export const wizardStore = {
     getStepIndex: () => _stepIndex,
     setStepIndex: (i: number) => { _stepIndex = i; _listeners.forEach(l => l()); },
     getCompletedSteps: () => _completedSteps,
     setCompletedSteps: (s: number[]) => { _completedSteps = s; _listeners.forEach(l => l()); },
     subscribe: (listener: Listener) => { _listeners.add(listener); return () => { _listeners.delete(listener); }; },
   };
   ```

   This store is consumed via `useSyncExternalStore` in both Sidebar and InterviewPage. The InterviewPage writes to it (setStepIndex, setCompletedSteps), and the Sidebar reads from it reactively.

2. **Modify `src/components/Sidebar.tsx`** to conditionally render wizard step list:

   Read wizard state reactively:
