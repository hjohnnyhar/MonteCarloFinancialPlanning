# Phase 2 — UI Design Contract: Interview Wizard

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — raw HTML elements styled with Tailwind v4 classes |
| Icon library | none — use Unicode characters or inline SVG for directional affordances (chevrons, checkmarks) |

**Decision source:** `<design_system_decision>` block — user explicitly chose no shadcn/ui; raw Tailwind v4 only.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding, gap between label and input |
| sm | 8px | Compact element spacing, gap between stacked form fields |
| md | 16px | Default element padding (inputs, buttons, cards), section content gap |
| lg | 24px | Section padding, wizard step body padding |
| xl | 32px | Layout gaps between major wizard sections |
| 2xl | 48px | Major section breaks, spacing between progress bar and form content |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for Back/Next buttons: minimum 44px height (accessibility — WCAG 2.5.5 target size)
- Progress bar height: 8px (fixed, not on scale)
- Sidebar width: 224px (w-56, inherited from Phase 1 shell — source: Sidebar.tsx)
- TopNav height: 56px (h-14, inherited from Phase 1 shell — source: TopNav.tsx)

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-6` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium leading-snug` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold leading-tight` |

Notes:
- Display (24px/bold) is used for the wizard step title only (e.g., "Income & Expenses")
- Heading (20px/semibold) is used for subsection titles within a step (e.g., "Monthly Expenses")
- Label (14px/medium) is used for all `<label>` elements above inputs and for secondary metadata
- Body (16px/regular) is used for all input placeholder text, help text, and paragraph content
- Font family is Inter throughout — inherited from `body` via layout.tsx className

## Color

All values are Tailwind v4 default palette tokens, consistent with Phase 1 components (source: AppShell.tsx, Sidebar.tsx, TopNav.tsx, page.tsx).

| Role | Tailwind Token | Hex (approx) | Usage |
|------|----------------|--------------|-------|
| Dominant (60%) | white / `bg-white` | #ffffff | Main content area background, input backgrounds, card backgrounds |
| Secondary (30%) | gray-50 / `bg-gray-50` | #f9fafb | Sidebar, progress step list background, review summary cards, disabled field backgrounds |
| Accent (10%) | blue-600 / `bg-blue-600` | #2563eb | Active/primary buttons (Next, Save), progress bar fill, active step indicator |
| Destructive | red-700 / `text-red-700` | #b91c1c | Inline validation error text, remove-goal button label |

Accent reserved for:
- "Next" navigation button (background)
- "Save & Continue" button (background)
- Progress bar fill color
- Active step indicator dot/number
- Focus ring on form inputs (`ring-blue-500`)

Accent is NOT used for: back buttons, secondary actions, headers, body text, borders, or hover states.

Supporting semantic tokens (not accent):
- Borders: `border-gray-200` (#e5e7eb) — all input borders, card borders, dividers
- Hover surface: `hover:bg-gray-100` (#f3f4f6) — nav items, step list hover
- Body text: `text-gray-900` (#111827) — all primary text
- Secondary text: `text-gray-500` (#6b7280) — help text, step descriptions, labels below inputs
- Error surface: `bg-red-50` with `border-red-200` and `text-red-700` — inline validation error containers
- Success/complete: `text-green-600` (#16a34a) — completed step checkmark only

## Component Contract

No component library is installed. Each component is raw HTML styled with Tailwind v4. These are the canonical class patterns — executor must use these exactly.

### Input (text, currency, percentage)

```html
<input
  type="text"
  inputMode="decimal"
  class="w-full rounded-md border border-gray-300 bg-white px-3 py-2 text-base text-gray-900 placeholder:text-gray-400 focus:border-blue-500 focus:outline-none focus:ring-2 focus:ring-blue-500/20"
/>
```

Currency fields: `type="text" inputMode="decimal"` — never `type="number"`. Parse with `z.coerce.number()` (source: RESEARCH.md Pitfall 5).

Error state adds: `border-red-500 focus:border-red-500 focus:ring-red-500/20`

### Label

```html
<label class="block text-sm font-medium text-gray-700">
  Salary
</label>
```

Always `block` (stack above input). Pair with `<p class="mt-1 text-sm text-gray-500">` for help text below.

### Field Group (Label + Input + Error)

```html
<div class="flex flex-col gap-1">
  <label class="block text-sm font-medium text-gray-700">Annual Salary</label>
  <input type="text" inputMode="decimal" class="..." />
  <p class="text-sm text-red-600">Must be 0 or greater</p>  <!-- only when error -->
</div>
```

### Button — Primary (Next / Save)

```html
<button
  type="submit"
  class="rounded-md bg-blue-600 px-4 py-2.5 text-sm font-semibold text-white hover:bg-blue-700 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50"
>
  Next
</button>
```

Minimum height: 44px enforced by `py-2.5` (10px top + 10px bottom) + `text-sm` line height.

### Button — Secondary (Back)

```html
<button
  type="button"
  class="rounded-md border border-gray-300 bg-white px-4 py-2.5 text-sm font-semibold text-gray-700 hover:bg-gray-50 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2"
>
  Back
</button>
```

### Button — Destructive (Remove Goal)

```html
<button
  type="button"
  class="text-sm font-medium text-red-600 hover:text-red-700 focus:outline-none focus:underline"
>
  Remove
</button>
```

No background — text-only destructive to avoid visual noise in goal list.

### Select

```html
<select
  class="w-full rounded-md border border-gray-300 bg-white px-3 py-2 text-base text-gray-900 focus:border-blue-500 focus:outline-none focus:ring-2 focus:ring-blue-500/20"
>
  <option value="">Select goal type...</option>
</select>
```

### RadioGroup (Risk Tolerance questionnaire)

One `<fieldset>` per question. Options stack vertically.

```html
<fieldset class="flex flex-col gap-3">
  <legend class="text-sm font-medium text-gray-700">When do you expect to need most of this money?</legend>
  <label class="flex items-center gap-3 cursor-pointer">
    <input
      type="radio"
      name="time_horizon"
      value="1"
      class="h-4 w-4 border-gray-300 text-blue-600 focus:ring-blue-500"
    />
    <span class="text-base text-gray-900">Less than 2 years</span>
  </label>
</fieldset>
```

Checked state uses browser default with `accent-blue-600` (Tailwind v4 supports `accent-{color}`).

### Progress Bar

```html
<div class="h-2 w-full rounded-full bg-gray-200">
  <div
    class="h-2 rounded-full bg-blue-600 transition-all duration-300"
    style="width: {percent}%"
  ></div>
</div>
```

Height: 8px. Transition: 300ms ease. Never animated on initial mount.

### Card / Panel (Review step summary cards, goal list items)

```html
<div class="rounded-lg border border-gray-200 bg-gray-50 p-4">
  <!-- content -->
</div>
```

Edit link within card:
```html
<button type="button" class="text-sm font-medium text-blue-600 hover:text-blue-700">Edit</button>
```

### Step List (Sidebar progress — wizard navigation)

Active step:
```html
<div class="flex items-center gap-3 rounded-md bg-blue-50 px-3 py-2">
  <span class="flex h-6 w-6 items-center justify-center rounded-full bg-blue-600 text-xs font-semibold text-white">1</span>
  <span class="text-sm font-medium text-blue-700">Income &amp; Expenses</span>
</div>
```

Completed step (adds checkmark, allows click navigation):
```html
<button class="flex w-full items-center gap-3 rounded-md px-3 py-2 hover:bg-gray-100">
  <span class="flex h-6 w-6 items-center justify-center rounded-full bg-green-100 text-xs text-green-700">&#10003;</span>
  <span class="text-sm text-gray-700">Income &amp; Expenses</span>
</button>
```

Future step (non-interactive):
```html
<div class="flex items-center gap-3 rounded-md px-3 py-2 opacity-50 cursor-not-allowed">
  <span class="flex h-6 w-6 items-center justify-center rounded-full border border-gray-300 text-xs text-gray-400">2</span>
  <span class="text-sm text-gray-400">Assets &amp; Liabilities</span>
</div>
```

## Layout Contract

### Wizard Shell Layout

The interview page uses the existing AppShell (TopNav + Sidebar). The wizard content replaces `<main>`.

```
[TopNav h-14]
[Sidebar w-56] [Wizard Content flex-1]
               ┌─────────────────────────────────────────┐
               │ Step Progress Bar (full width, top)      │
               │ [Step 1 of 5] ████████░░░░░░░░ 40%      │
               ├─────────────────────────────────────────┤
               │ Step Title (display, 24px bold)          │
               │ Step Description (body, gray-500)        │
               │                                         │
               │ Form content (max-w-2xl, mx-auto)        │
               │                                         │
               │ ─────────────────────────────────────── │
               │ [Back]                        [Next →]   │
               └─────────────────────────────────────────┘
```

- Wizard content area: `flex-1 overflow-auto bg-white`
- Inner form container: `max-w-2xl mx-auto px-6 py-8`
- Nav button row: `flex items-center justify-between mt-8 pt-6 border-t border-gray-200`
- Progress bar strip: `w-full px-6 py-4 border-b border-gray-200 bg-white`

### Goals Step — Inline Expand/Collapse

Goal list item is a card. "Add Goal" expands a form below the list. No modal dependency (source: RESEARCH.md Open Question 3 recommendation).

```
[Goal List]
  [Goal Card: Retirement — $2M target]  [Edit] [Remove]
  [Goal Card: Education — $100K target]  [Edit] [Remove]
  [+ Add Goal ▼]
    ┌──────────────────────────────────┐
    │ Goal Type: [Select ▼]            │
    │ [type-specific fields expand]    │
    │ [Cancel]          [Save Goal]    │
    └──────────────────────────────────┘
```

### Review Step — Read-Only Summary

Each wizard section renders as a summary card with an "Edit" link that navigates back to that step index.

```
[Section: Income & Expenses]  [Edit]
  Salary: $120,000/yr
  Monthly spend: $4,500

[Section: Assets & Liabilities]  [Edit]
  ...

[Section: Financial Goals]  [Edit]
  3 goals defined

[Section: Risk Tolerance]  [Edit]
  Moderate (score: 7/10)

[Continue to Simulation →]
```

## Accessibility Contract

- All `<input>` and `<select>` elements have an associated `<label>` via `for`/`id` pairing.
- All `<fieldset>` groups (radio questions) have a `<legend>`.
- Error messages are linked to inputs via `aria-describedby`.
- Invalid inputs carry `aria-invalid="true"` when in error state.
- The step navigation list uses `aria-label="Wizard progress"` on the container `<nav>`.
- Active step has `aria-current="step"`.
- Completed steps that are buttons have descriptive labels: `aria-label="Edit Income & Expenses, step 1"`.
- Progress bar carries `role="progressbar" aria-valuenow={percent} aria-valuemin="0" aria-valuemax="100"`.
- Destructive "Remove" confirmation uses `role="alertdialog"` on the inline confirm container.
