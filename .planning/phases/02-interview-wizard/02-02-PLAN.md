   ```typescript

   ```

   This page:
   - Calls `usePlan()` to get plan data, isLoading, updatePlan
   - Manages `stepIndex` state initialized from `plan?.metadata?.wizardStep ?? 0`
   - Syncs stepIndex to `wizardStore` so Sidebar picks up wizard state
   - Renders loading state: `<div className="flex items-center justify-center p-12 text-sm text-gray-400">Loading your plan...</div>` (copy from UI-SPEC)
   - Renders WizardShell with the active step component

   Step rendering: Use a switch statement on stepIndex. Each step receives `plan`, `onComplete`, and `onBack` props. The step owns its Back/Next buttons internally.

   ```typescript
   const stepConfig = WIZARD_STEPS[stepIndex];

   // handleStepComplete is passed to each step
   const handleStepComplete = async (stepData: Partial<FinancialPlan>) => {
     const nextIndex = stepIndex + 1;
     await updatePlan({ ...stepData, metadata: { wizardStep: nextIndex } });
     setStepIndex(nextIndex);
     // Update completed steps in wizardStore
     const newCompleted = [...new Set([...wizardStore.getCompletedSteps(), stepIndex])];
     wizardStore.setCompletedSteps(newCompleted);
   };

   const handleBack = () => {
     setStepIndex(s => Math.max(0, s - 1));
   };
   ```

   Resume logic (INT-06): `useEffect` that reads `plan?.metadata?.wizardStep` on first load and sets stepIndex. Also compute completedSteps: all indices < savedWizardStep are considered completed.
   ```typescript
   useEffect(() => {
     if (plan && !hasResumed.current) {
       const saved = plan.metadata.wizardStep ?? 0;
       setStepIndex(saved);
       // Mark all steps before saved as completed
       const completed = Array.from({ length: saved }, (_, i) => i);
       wizardStore.setCompletedSteps(completed);
       hasResumed.current = true;
     }
   }, [plan]);
   ```

   Pre-population (INT-08): Each step receives `plan` object and uses its values as `defaultValues` in useForm. This happens automatically since usePlan loads from server.

   Sync stepIndex with wizardStore so Sidebar updates:
   ```typescript
   useEffect(() => {
     wizardStore.setStepIndex(stepIndex);
   }, [stepIndex]);
   ```

   Also listen for Sidebar clicks (bidirectional sync): The Sidebar calls `wizardStore.setStepIndex()` when a completed step is clicked. The InterviewPage reads this via useSyncExternalStore:
   ```typescript
   import { useSyncExternalStore } from 'react';
   const externalStepIndex = useSyncExternalStore(wizardStore.subscribe, wizardStore.getStepIndex, () => 0);
   useEffect(() => {
     setStepIndex(externalStepIndex);
   }, [externalStepIndex]);
   ```

Per D-04 (warn-but-allow), each step uses this pattern:
- Use `useForm` with `zodResolver` for error message display
- The Next button calls `handleNext` which: (a) calls `trigger()` to validate and show inline warnings, (b) calls `getValues()` and passes values to `onComplete()` regardless of validation result
- This means validation warnings display inline but never block advancement

1. **Create `src/components/interview/steps/IncomeExpensesStep.tsx`:**

   ```typescript
   'use client';
   import { useForm } from 'react-hook-form';
   import { zodResolver } from '@hookform/resolvers/zod';
   import { useEffect } from 'react';
   import { incomeExpensesSchema, type IncomeExpensesFormData } from '@/lib/wizardSchemas';
   import type { FinancialPlan } from '@/lib/types';
   ```

   Props: `{ plan: FinancialPlan; onComplete: (data: Partial<FinancialPlan>) => Promise<void>; onBack: (() => void) | null; }`

   Form setup:
   ```typescript
   const { register, formState: { errors }, trigger, getValues, reset } = useForm<IncomeExpensesFormData>({
     resolver: zodResolver(incomeExpensesSchema),
     defaultValues: {
       income: { salary: 0, otherAnnualIncome: 0, annualSavingsRate: 0 },
       expenses: { monthlyEssential: 0, monthlyDiscretionary: 0, monthlyDebtPayments: 0 },
     },
   });

   // Reset when plan loads (Pitfall 2 fix)
   useEffect(() => {
     if (plan) {
       reset({ income: plan.income, expenses: plan.expenses });
     }
   }, [plan, reset]);
   ```

   handleNext function (D-04 warn-but-allow):
   ```typescript
   const handleNext = async () => {
     await trigger(); // Display validation warnings
     const values = getValues();
     await onComplete({ income: values.income, expenses: values.expenses });
   };
   ```
